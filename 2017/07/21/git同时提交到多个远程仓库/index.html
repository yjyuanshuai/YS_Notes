git push 多个远程仓库

2017-07-21
Git
Git  版本控制
  • 前提
  • 步骤
    • 在 csdn 和 码云 上分别建仓库
    • 关联远程仓库
    • pull/push 测试
  • SourceTree 实现一次推送到 2 个远程仓库
    • 自动同时推送到远程
    • 手动选择推送

前提

将同一份文档传到 2 个代码托管平台，防止文档丢失。

步骤

  1. 建远程仓库
  2. 本地文件夹关联远程仓库
  3. pull/push 测试

在 csdn 和 码云 上分别建仓库

git@git...csdn

git@git...oschina

关联远程仓库

git remote add guroad(自定义) git@git...csdn

git remote set-url --add guroad(保持一致) git@git...oschina

pull/push 测试

git pull guroad --all

git push guroad --all

SourceTree 实现一次推送到 2 个远程仓库

自动同时推送到远程

设置 .git/config 文件内容，如下：

[core]
    repositoryformatversion = 0
    filemode = false
    bare = false
    logallrefupdates = true
    symlinks = false
    ignorecase = true
    hideDotFiles = dotGitOnly
[remote "origin"]
    url = https://git.oschina.net/shuguang/cmsdev.git
    url = https://git.coding.net/bagesoft/cmsdev.git
    fetch = +refs/heads/*:refs/remotes/origin/*
[branch "master"]
    remote = origin
    merge = refs/heads/master

手动选择推送

[core]
    repositoryformatversion = 0
    filemode = false
    bare = false
    logallrefupdates = true
    symlinks = false
    ignorecase = true
    hideDotFiles = dotGitOnly
[remote "origin"]
    url = https://git.oschina.net/shuguang/cmsdev.git
    fetch = +refs/heads/*:refs/remotes/origin/*
[remote "coding"]
    url = https://git.coding.net/bagesoft/cmsdev.git
    fetch = +refs/heads/*:refs/remotes/origin/*
[branch "master"]
    remote = origin
    merge = refs/heads/master
