github+jekyll搭建个人博客

需求

  • 简介的界面，有分类、标签功能，能够按时间顺序排列，看到自己哪天都记录了些什么；

  • 界面、域名自定义；

  • 管理方便；

这个阶段在网上查了较多资料，因为没有网页开发的经验，考虑到学习成本，选择了 github+jekyll 的方式来实现。

注意

Github 主页虽然有很多好处，但也有相应的缺陷。如：

开源，注意隐私保护和敏感数据；

生成的是静态网页，动态功能需要依靠外部服务器，比如评论功能需要disqus等；

没有用到数据库，每次运行都遍历全部的文本文件，不适合大型网站。

准备

安装/更新 Gem、jekyll 版本，没 Github 账号的要注册一个。

了解 Github Pages 与 jekyll。

实践

采用项目主页，即新建项目仓库，创建 gh-pages 分支的方法，实现主页博客。

以下为首次实现时的过程记录，个人认为实际用起来不够简便，后面根据自己的需要继续整理修改。

1. 新建项目仓库

  • 登录 Github 账号，创建项目仓库 YS_Notes；

  • 创建 gh-pages 分支，克隆仓库到本地；

2. 本地操作

  • 自己写一个或者找一个符合自己需求的 jekyll theme，还可以 fork 其他人的。此处感谢 Github-HyG 提供的主题；

整理

阶段一，初始功能实现

一个仓库，两个分支。master分支放源码，gh-pages分支放生成的主页文件。

  • 将网站源码放在 YS_Notes master 分支上；

  • 每次本地测试（jekyll serve）后，将_site拷到本地tmp中；

  • 再切换到YS_Notes gh-pages分支，将tmp内容复制进去，上传到远程gh-pages分支。

阶段二，使用 makefile 简化阶段一种的步骤

过程同阶段一，区别是将手动过程用 makefile 文件代替了。在本地YS_Notes仓库下执行 make makefile命令。

makefile 内容：

deploy:
	git checkout master
	jekyll build
	git add -A
	git commit -m "update master"
	cp -r _site/ /tmp/
	git checkout gh-pages
	rm -r ./*
	cp -r /tmp/_site/* ./
	git add -A
	git commit -m "deploy blog"
	git push origin gh-pages
	git checkout master
	echo "deploy succeed"
	git push origin master
	echo "push master"

阶段三，分开管理

用两个仓库来管理，一个用于放源码，另一个用于放最终生成的主页。

  • 创建2个空的远程仓库YS_Notes_Source（源码）、YS_Notes（主页）；

  • YS_Notes 新建分支，分支名必须为 gh-pages；

  • 克隆 YS_Notes_Source至本地 YS_Blog，在 YS_Blog 中创建 2 个子目录 source、finnal_site；

  • source 中存放源码，finnal_site 关联远程仓库 YS_Notes；

  • jekyll 测试或生成时不能直接使用 jekyll serve，而要加上源和目的地址jekyll serve [--watch] --source source (YS_Notes_Source中的子目录) --destination finnal_site (YS_Notes_Source中的子目录)；

  • 这样修改源码后，上传至 YS_Notes_Source，本地测试主页后，上传至 YS_Notes。
