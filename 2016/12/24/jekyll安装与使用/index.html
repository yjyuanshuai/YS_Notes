安装

1、安装命令

安装之前，查看 ruby 和 gem 版本，更新至最高版本


// 10.11 前
sudo gem install jekyll

// 10.11 及以后
sudo gem install -n /usr/local/bin jekyll

基本用法

1、build

jekyll build

// 当前文件夹中的内容将会生成到 ./site 文件夹中。

jekyll build --destination <destination>

// 当前文件夹中的内容将会生成到目标文件夹<destination>中。

jekyll build --source <source> --destination <destination>

// 指定源文件夹<source>中的内容将会生成到目标文件夹<destination>中。

jekyll build --watch
// 当前文件夹中的内容将会生成到 ./site 文件夹中，查看改变，并且自动再生成。

2、使用开发服务器，预览

jekyll serve  //(--source <source> --destination <destination>)

本地浏览器访问 http://localhost:4000 或 127.0.0.1:4000，是否正常访问

jekyll serve --detach

# => 功能和`jekyll serve`命令相同，但是会脱离终端在后台运行。
#    如果你想关闭服务器，可以使用`kill -9 1234`命令，"1234" 是进程号（PID）。
#    如果你找不到进程号，那么就用`ps aux | grep jekyll`命令来查看，然后关闭服务器。[更多](http://unixhelp.ed.ac.uk/shell/jobz5.html).

jekyll serve --watch

# => 和`jekyll serve`相同，但是会查看变更并且自动再生成。

问题

1. 安装时

ERROR:  Error installing jekyll:
    ERROR: Failed to build gem native extension.

安装 Command Line Tools

xcode-select --install

2. 使用中

jekyll serve

Dependency Error: Yikes! It looks like you don't have XXX or one of its dependencies installed.

类似这种错误，安装对应的 XXX 即可。如 jekyll-paginate，bundler等。

sudo gem install -n /usr/local/bin bundler

目录结构

文件/目录 含义 语法
_config.yml 全局配置  
_drafts 草稿  
_includes 文章中可以重用的部分，可以放这里  
_layouts 各种文章外部的模板（可以在文章头信息中选择，layout:XXX ）  
_posts 所有的文件（有一定的格式，后缀有 md/textile） yyyy-MM-dd-文章名.md
_data    
_site 生成的可视化的主页  
index.html 站点的首页，整个站的入口  
_plugins 插件放置的目录  
自定义目录 含义
sitemap.txt  
_assets 用到的资源文件，如图片
