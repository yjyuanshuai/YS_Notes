全终端操作

前提：有 ruby、rubyGems

准备

设置 ruby 源

天朝墙了原本的亚马逊 https://rubygems.org/ 源，所以需更改。

1、查看当前的 ruby 源

gem sources -l

2、添加可用的 ruby 源（目前是：https://gems.ruby-china.org）

gem gem sources -a 有效的ruby源

3、删除旧的

gem sources --remove 无效的ruby源

4、使用上述查看ruby源的命令，查看是否成功，确保只有一个源

安装 CocoaPods

1、开始安装

// OS X 10.11 前
sudo gem install cocoapods

// OS X 10.11 及之后
sudo gem install -n /usr/local/bin cocoapods

安装完成，最后一条信息是：gems installed

2、判断 CocoaPods 是否可用

pod setup

经常会出现卡在 Setting up CocoaPods master repo 的情况，使用以下方法可以查看下载进度：

// 终端输入：cd ~/.cocoapods，进入cocoapods
du -sh *

3、常见错误

经常会有卡在 Setting up CocoaPods master repo 很久后报以下错：

[!] /usr/bin/git clone https://github.com/CocoaPods/Specs.git master

Cloning into 'master'...
error: RPC failed: result=56, HTTP code = 200
fatal: the remote end hung up unexpectedly
fatal: early EOF
fatal: index-pack failed

可能的原因有 3 个：

  • gem 版本过低 
    sudo gem update --system
  • github 无法连接 
    ping github.com
  • .cocoapods目录下的配置信息错误 
    pod repo list

手动下载 CocoaPods 库安装

更新记录

2017.11.24

Mac OS X 10.12.6
gem 2.6.14

执行sudo gem install -n /usr/local/bin cocoapods后，cd ~/.cocoapods/repos找不到路径。

步骤

1、检查是否有以下路径存在

cd ~/.cocoapods/repos

若没有，执行以下命令（猜测手动建路径应该也可以）

sudo gem install cocoapods

有路径后，手动下载/克隆 Specs 库

  • 在 ~/.cocoapods/repos 路径下，克隆 Specs 库 
    git clone https://github.com/CocoaPods/Specs
  • 手动下载 https://github.com/CocoaPods/Specs

2、调整文件名和文件层次

更改上述 Specs 库名为 master

将新改名的 master 文件转移到 ~/.cocoapods/repos 下

可能的问题

运行pod后显示：”pod: command not found”的错误。赋予/usr/local/bin给予执行与读取权限，执行以下命令。若不行执行sudo chmod 777 /usr/local/bin/。

sudo chmod 700 /usr/local/bin/
