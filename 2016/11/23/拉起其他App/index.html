URL Scheme 拉起 App 实例

拉起别人的 App

基本流程：

1、确认其他 App 的 URL Scheme;
2、调用 canOpenUrl，对不可打开的情况给出提示。可打开的调用 openUrl。

注：iOS 9 后需要添加 URL Scheme 白名单，才能使用 canOpenURL 来判断，白名单最多 50 个。

示例

//先判断再打开WXApp
- (void)demo2 {
    //创建一个url，这个url就是WXApp的url，记得加上：//
    NSURL *url = [NSURL URLWithString:@"weixin://"];

    //先判断是否能打开该url
    if ([[UIApplication sharedApplication] canOpenURL:url]) {
        //打开url
        [[UIApplication sharedApplication] openURL:url];
    }else {
        //给个提示或者做点别的事情
        NSLog(@"U四不四洒，没安装WXApp，怎么打开啊！");
}

添加白名单

点击info.plist->右键->Open As->Source Code->添加下面的代码

<key>LSApplicationQueriesSchemes</key>
    <array>
        <string>weixin</string>
    </array>

让其他应用拉起自己的 App

基本的流程：

1、在工程中注册 URL Scheme；
2、其他应用，通过 openUrl: 方法，即可拉起 App；
3、AppDelegate 代理方法，处理拉气后的操作。

注册 URL Scheme

添加过 URL Scheme 后，info.plist 中可以看到 URL types 组中即为添加的内容。

代理方法

共有 3 个代理方法，分别对应不同的系统，有优先级。

// 2.0 - 9.0 有效，优先级最低
- (BOOL)application:(UIApplication *)application handleOpenURL:(NSURL *)url NS_DEPRECATED_IOS(2_0, 9_0, "Please use application:openURL:options:") __TVOS_PROHIBITED;


// 4.2 - 9.0 有效，优先级次之
- (BOOL)application:(UIApplication *)application openURL:(NSURL *)url sourceApplication:(nullable NSString *)sourceApplication annotation:(id)annotation NS_DEPRECATED_IOS(4_2, 9_0, "Please use application:openURL:options:") __TVOS_PROHIBITED;

// 9.0 - 后有效，优先级最高
- (BOOL)application:(UIApplication *)app openURL:(NSURL *)url options:(NSDictionary<NSString*, id> *)options NS_AVAILABLE_IOS(9_0);
