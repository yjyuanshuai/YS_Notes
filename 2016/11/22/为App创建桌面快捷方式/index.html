为 App 创建桌面快捷方式

基本原理

程序内部创建一个简单的 Web 站点，通过这个站点调用 Safari，站点将自定义的Html页面返回给 Safari，此时利用 Safari 的“添加至主屏幕”功能，将自定义的 Html 制作成桌面书签，当用户点击桌面图标时，则运行自定义的 Javascript 来进行跳转至App。

涉及知识

  • Javascript
  • Data URI Scheme
  • Socket 基本知识（CocoaHTTPServer-第三方HTTPServer库）
  • Base64

1、Data URI Scheme

用于内嵌一些小的图片等内容，减少 HTTP 请求。

URI 语法：

<img src="data:image/png;base64;iVB0...">

语法分析：

data: 取数据协议名称，此处为 image/png
image/png: 数据类型名称
base64: 数据的编码方法
iVBOR...: 编码后的数据

2、Base64

一种编码方式，把 8-bit 数据翻译成标准 ASCII 字符。

1、创建 URL Scheme

2、核心代码

- (IBAction)action:(id)sender
{
    [DDLog addLogger:[DDTTYLogger sharedInstance]];
    _httpServer = [[HTTPServer alloc] init];
    [_httpServer setType:@"_http._tcp."];
    NSString *webPath = [[[NSBundle mainBundle] resourcePath] stringByAppendingPathComponent:@"Web"];
    DDLogInfo(@"Setting document root: %@", webPath);
    [_httpServer setDocumentRoot:webPath];
    [self startServer];
}


- (void)startServer
{
    // Start the server (and check for problems)

    NSError *error;
    if([_httpServer start:&error])
    {
        DDLogInfo(@"Started HTTP Server on port %hu", [_httpServer listeningPort]);
        // open the url.
        NSString *urlStrWithPort = [NSString stringWithFormat:@"http://localhost:%d",[_httpServer listeningPort]];
        [[UIApplication sharedApplication] openURL:[NSURL URLWithString:urlStrWithPort]];
    }
    else
    {
        DDLogError(@"Error starting HTTP Server: %@", error);
    }
}

3、html 文件

放在文件夹 Web 目录下，切记这个文件在工程中是 实体文件夹，folder references

  • index.html
  • content.html（实际上是过度文件，不需要放到项目中）

index.html

<!DOCTYPE html>
<html>
<head>
    <meta http-equiv="Content-Type" content="text/html; charset=utf-8">
    <meta http-equiv="refresh" content="0;URL=...">
</head>
</html>

语法分析：

<meta http-equiv="refresh" content="0;URL=...">

定期刷新页面，content 后是刷新的间隔时间（秒），URL 是刷新后重定向的页面。

此处...是我省略的，实际上应该是 content.html base64编码后的内容。

content.html

<!DOCTYPE html>
<html>
<head>
    <meta name="apple-mobile-web-app-capable" content="yes">
    <meta name="apple-mobile-web-app-status-bar-style" content="black">
    <meta content="text/html charset=UTF-8" http-equiv="Content-Type"/>
    <meta name="viewport" content="width=device-width,initial-scale=1.0,user-scalable=no"/>
    <link rel='apple-touch-icon' href='data:image/jpeg;base64,...'>
    <title>YS_Test快捷方式</title>
</head>
<body>
    <a href="YSDevScheme://" id="qbt" style="display:none"></a>
    <span id="msg"></span>
</body>
<script>
    if (window.navigator.standalone == true)
    {
        var lnk = document.getElementById("qbt").click();
    }
    else
    {
        document.getElementById("msg").innerHTML='<div style="font-size:12px">这里可以添加引导页面</div>';
    }
</script>
</html>

内容分析：

1）link-图标

<link rel='apple-touch-icon' href='data:image/jpeg;base64,...'>

... 代指的是快捷方式的图标，经过 base64编码后的内容。

2）title-标题

<title>YS_Test快捷方式</title>

桌面快捷方式的标题。

3）body-URL Scheme

span标签用于占位，具体内容由script决定。

4）script-添加快捷方式

首先判断 Safari 是否是全屏显示，（如：点击已存在的桌面快捷方式，Safari 就会以全屏显示）。

非全屏显示，就可以提示用户添加桌面快捷方式。
