背景简介

开发过程中常常需要使用第三方开源库，而最大的开源库平台非 Github 莫属。

DZNEmptyDataSet主要是iOS开发中用于无数据时展示空白背景的第三方开源库，最近更新的 1.8.1 版本中出现一个问题，自定义背景custemView中若有Butten，没有点击效果，问题链接。切换至 1.6.1 版本没有该问题，但是会出现另一个问题，tableView 有 headerView 且 headerView 上有操作按钮时，空白背景会遮挡 headerView 导致无法点击。

这里，记录下我第一次参与开源库。

参与开源库

前提：

github 账号。

基本的 github 使用，如 ssh、push、fork等。

创建话题

如，发现了一个bug，可以创建一个issue 和其他人一起讨论。

创建 issue 注意以下几点：issue guides

  • 检查是否已经有这个 issue 存在，没有则创建一个，有就不要重复创建了；
  • 务必对自己的问题有清晰的认识：期望的结果是什么？然而却发生了什么？ 详细描述其他人如何重现该问题；
  • 包含一些系统相关的细节，比如用的什么浏览器、库或操作系统及版本号；
  • 在你的话题或在 Gist 里贴出你的错误输出或日志。如果在话题里贴出来，请用三个反引号包围起来使得能够良好的呈现给大家；
  • 在像 JSFiddle 或 CodePen 类似的平台上重现该问题并给出问题demo的链接。

Pull Request

若发现一个bug，并且可以修复它。想把修改后的代码提交给原作者，为开源库做些贡献，则需要 pull request。

因为别人的开源库，我是没有修改权限的（除非作者把我加入到协作者，当然这是极少数大牛才有可能），所以能做的就是，把别人的开源库 fork 到自己的账号下。然后操作自己账号的的同名仓库（如：这里是 DZNEmptyDataSet ），既要保证可以随时更新这个开源库，又要保证自己修改的部分是独立的。以下以 DZNEmptyDataSet 为例：

  • fork DZNEmptyDataSet (此处为了区别将我fork后的仓库记为 yj/DZNEmptyDataSet )；
  • clone yj/DZNEmptyDataSet 至本地；
  • 将本地仓库关联原 DZNEmptyDataSet，将其添加为一个远程仓库；
  • 创建新分支，用于添加自己的修改；
  • push 到 yj/DZNEmptyDataSet；
  • pull request。

注：clone、remote add 都要用 ssh，而非 http.

fork 仓库

找到原第三方库，将其 fork 到自己账号下，成功后我自己的账户下会有一个同名的仓库（这里为了区别，将其记为 yj/DZNEmptyDataSet，实际是DZNEmptyDataSet）。

之后对代码的修改都是 push 到yj/DZNEmptyDataSet，因为原仓库没权限。

clone

git clone git@github.com:yjyuanshuai/DZNEmptyDataSet.git

remote add

注：要在本地仓库地址下，进行下面操作。

复制原作者的项目地址，添加为自己的一个远程仓库，用来实时将原项目的修改更新到咱们本地并合并。

// upstream 是自己写的，写成其他任何字符串皆可
git remote add upstream git@github.com:dzenbot/DZNEmptyDataSet.git

创建 branch

这只是一个约定成俗的方式，当然你也可以在master上添加修改，创建新的branch添加自己的修改的好处是，你可以同时添加多个修改，在一个修改还没有被原作者merge时，你可以用master创建新的branch继续你的其它修改。

git branch issue376
git checkout issue376

然后在这个新分支上修改代码，push 到 yj/DZNEmptyDataSet。

pull request

到我的 github 上，点击 Compare & pull request，向原作者请求合并到原项目中。

也可以选择issue376分支，点击右侧pull request。
