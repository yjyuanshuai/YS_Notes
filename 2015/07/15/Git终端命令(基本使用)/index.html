基本使用

仓库

1. 从本地关联远端仓库

git remote add origin https://github.com/username/reponame.git

// 首次推送 master 分支所有内容
git push -u origin master

2. 克隆远端仓库至本地

git clone git@github.com:michaelliao/gitskills.git

3. 建立本地分支与远程分支的联系

git branch --set-upstream-to=origin/branch-name branch-name

可能遇到的问题：

 “fatal: refusing to merge unrelated histories”

解决：

git pull origin master --allow-unrelated-histories

分支

1. 创建分支

创建并切换到分支

git checkout -b 新建分支名

相当于：

git branch 新建分支
git checkout 新建分支

2. 查看分支

会列出所有分支，当前分支前用 * 表示

git branch

3. 切换分支

git checkout 分支名

4. 合并分支

git merge 想要合并进当前分支的分支

5. 删除分支

git branch -d 将被删除的分支

强行删除一个未被合并的分支：

git branch -D 将被删除的分支

标签

1. 查看标签

git tag

2. 添加标签

git tag 标签名

git tag -a 标签名 -m "备注信息"

3. 删除标签

git tag -d 标签名

git push origin --tags

注：我在 coding 上测试的私有仓库，上面的方法无法删除标签，要用下面的才能删除。

git push origin --delete tag "标签名"

推送/下拉

1. commit

git init    //初始化git仓库（已存在，则省略）
git add 被修改的文件（. 或 * 表示全部文件）
git commit -m "备注的信息"

2. pull

// 默认下拉对应分支
git pull

// 下拉
git pull origin（remote名） 目标分支

3. push

// 首次推送
git push -u origin（仓库名） master（分支名）

// 推送到远程分支
git push origin（remote名） 目标分支

// 默认推送到当前对应分支
git push
