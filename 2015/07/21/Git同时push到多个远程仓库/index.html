需求

  • 同一份代码存在 2 个不同的远程仓库；
  • 一次 push，同时推送到 2 个仓库；

Github 有时访问速度过慢，为了及时的获取或推送到远程仓库，我希望将同一份代码同时存到 2 个不同的远程仓库，比如一个是 Github，一个是 coding，这样在一个仓库不给力的时候，可以及时用另一个仓库。

实现

通过配置 .git/config文件，可以实现我的需求。

配置完后，基本的信息如下：

[core]
	repositoryformatversion = 0
	filemode = true
	bare = false
	logallrefupdates = true
	ignorecase = true
	precomposeunicode = true
[remote "remote名"]
	url = 仓库一地址
	url = 仓库二地址
	fetch = +refs/heads/*:refs/remotes/remote名/*
[branch "master"]
	remote = remote名
	merge = refs/heads/master

在 github 和 coding 上分别建仓库

以下简称 git@github_repo1，和 git@coding_repo2。

关联远程仓库，修改配置文件

terminal 实现

打开 terminal，进入项目目录下。

git remote add remote名(自定义，如：guroad、origin等) git@github_repo1

git remote set-url --add remote名(与上面的保持一致) git@coding_repo2

注：上面 2 个 remote 名可以不同，这里保持相同，是为了实现一次推送到 2 个仓库。remote 名不同时，需分开推送。

直接打开配置文件修改

pull/push 测试

terminal

git pull guroad master

git push guroad --all

若remote 名是 origin，可省略，即：

git pull

git push (--all)

SourceTree

配置文件设置好后，正常推送即可。

拓展

一份代码关联 2 个远程仓库，每次只推送到一个指定远程仓库。修改文件内容 .git/config如下：

[core]
    repositoryformatversion = 0
    filemode = false
    bare = false
    logallrefupdates = true
    symlinks = false
    ignorecase = true
    hideDotFiles = dotGitOnly
[remote "origin"]
    url = https://git.oschina.net/shuguang/cmsdev.git
    fetch = +refs/heads/*:refs/remotes/origin/*
[remote "coding"]
    url = https://git.coding.net/bagesoft/cmsdev.git
    fetch = +refs/heads/*:refs/remotes/origin/*
[branch "master"]
    remote = origin
    merge = refs/heads/master
